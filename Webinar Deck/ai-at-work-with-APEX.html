Demos — All in APEX

Anchor the value

  • Three live flows: triage, brief, policy Q&A

Demo A: Complaint Triage → Structured Summary → Reply

Steps

  • Paste raw email thread into APEX input field
  • Extract JSON: {"customer": "name", "issue": "description", "severity": "high/medium/low", "product": "name", "due_date": "YYYY-MM-DD", "evidence": "snippet"}
  • Critique via QA prompt: flag missing evidence or ambiguous severity
  • Rewrite empathetic customer reply (≤120 words) + internal ticket notes
  • Save JSON + notes to sheet/CRM via APEX connector

Success Criteria

JSON fields all populated; reply ≤120 words; ticket notes include root-cause guess + next step

Failure Handling

If severity missing → request clarification stub; if evidence absent → add TODO tag

Demo B: Executive Brief from Messy CSV

Steps

  • Upload CSV export to APEX
  • Normalize headers/types; return clean table + anomaly list
  • Generate 120-word executive summary + 1 chart description
  • Store artifacts; preview chart; attach source note

Success Criteria

Totals reconcile; anomalies labeled; risks mentioned first in summary

Demo C: Policy Q&A with Citations

Steps

  • Add 5 sanitized PDFs into an APEX "Knowledge" collection
  • Ask policy questions; require source snippets + page IDs in response
  • "No answer" route if confidence low or source unavailable

Success Criteria

Answers always include citations or abstain safely. No hallucinated policy statements.

Automation A: Inbox Triage & Reply

Trigger

New email arrives in support inbox

Inputs

Raw email + allowlist metadata (known products, SLA tiers)

Steps

  • Classify issue type (billing, technical, feature request)
  • Extract JSON (customer, issue, severity, product)
  • Draft reply using tone template
  • Human approval gate (Slack notification or APEX UI)
  • Send approved reply + log transaction

Outputs

Reply email, ticket notes, run log

Exceptions

Unknown product → route to queue; no SLA match → escalate to manager

Metrics

Time-to-first-response, first-contact resolution rate, approval rate

Automation B: Weekly Ops Report

Trigger

Every Friday 16:00

Inputs

CSV exports from ops tools (issues, deployments, incidents)

Steps

  • Clean and merge CSVs
  • Compute KPIs (mean time to resolve, deployment frequency, incident count)
  • Detect anomalies (threshold: >2 std dev)
  • Produce 120-word summary + anomaly list + chart description
  • Deliver to Slack channel and email distribution list

Outputs

KPI table, anomaly list, summary, chart preview

Metrics

Prep time saved (baseline: 90 min → 5 min), anomaly detection precision

Handout: Policy One-Pager

Do

  • Use approved APEX workspace only
  • Redact PII before pasting (names, emails, IDs)
  • Require citations for all factual claims
  • Log all high-stakes decisions (approval, rejection, escalation)
  • Review outputs before sending externally

Don't

  • Paste customer data into unapproved tools
  • Skip human review for customer-facing outputs
  • Ship outputs without source verification
  • Use AI for final financial/legal decisions without expert review

Incident Steps

If you suspect a data leak or wrong output was sent:

  1. Stop the automation immediately
  2. Notify your manager and security@company.com
  3. Document: what was sent, to whom, when, what data was included
  4. Follow incident response playbook (link: __________)

  • Offer 1:1 co-build sessions for first 2 automations
